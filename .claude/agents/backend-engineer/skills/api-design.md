# API 設計（API Design）

## 概要

RESTful API の設計・実装・ドキュメント整備を行うスキル。

## 責務

### エンドポイント設計

- リソース指向の URL 設計を行う
- 適切な HTTP メソッドとステータスコードを選定する
- ページネーション、フィルタリング、ソートの標準パターンを適用する

### バリデーション

- Zod スキーマによるリクエストバリデーションを実装する
- エラーレスポンスの形式を統一する
- 入力値のサニタイズを徹底する

### ドキュメント

- OpenAPI 仕様書を作成・更新する
- リクエスト・レスポンスの具体例を記載する
- 認証・認可の要件を明記する

## 出力フォーマット

```text
## API エンドポイント定義

- メソッド: GET/POST/PUT/DELETE
- パス: /api/v1/resource
- 認証: 必要/不要
- リクエスト: Zod スキーマ定義
- レスポンス: 正常系・異常系の型定義
- ステータスコード: 各ケースの対応表
```text
