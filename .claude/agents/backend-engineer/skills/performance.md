# パフォーマンス（Performance）

## 概要

バックエンドのパフォーマンスボトルネックを特定し、改善するスキル。

## 責務

### ボトルネック分析

- レスポンスタイム、スループットを計測する
- プロファイリングでホットスポットを特定する
- DB クエリのパフォーマンスを分析する

### キャッシュ戦略

- キャッシュすべきデータを選定する
- TTL とキャッシュ無効化戦略を設計する
- キャッシュヒット率を監視する

### 最適化

- 不要な DB アクセスを削減する
- バッチ処理・非同期処理の適用を提案する
- メモリ使用量を最適化する

## 出力フォーマット

```text
## パフォーマンス改善提案

- 対象: ボトルネック箇所
- 現状: 計測値（レスポンスタイム、スループット等）
- 原因: ボトルネックの根本原因
- 改善案: 具体的な施策
- 期待効果: 改善後の目標値
- リスク: 施策に伴うリスク
```text
