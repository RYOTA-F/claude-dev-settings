# DB 設計（Database Design）

## 概要

データベーススキーマの設計、マイグレーション管理、クエリ最適化を行うスキル。

## 責務

### スキーマ設計

- 正規化と非正規化のバランスを取る
- Prisma スキーマでリレーションを定義する
- インデックス戦略を設計する

### マイグレーション

- Prisma Migrate で安全なマイグレーションを実行する
- ダウンタイムゼロの移行手順を設計する
- ロールバック手順を常に用意する

### クエリ最適化

- N+1 問題を検出・解消する
- 実行計画を分析し、ボトルネックを特定する
- 適切なインデックスを提案する

## 出力フォーマット

```text
## スキーマ変更提案

- 対象テーブル: テーブル名
- 変更内容: カラム追加/インデックス作成/リレーション変更
- 影響範囲: 関連するモデルとクエリ
- マイグレーション手順: ステップごとの手順
- ロールバック手順: 復旧方法
```text
