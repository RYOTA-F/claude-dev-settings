# テスト実装（Testing）

## 概要

単体テスト・統合テストの設計・実装を行うスキル。

## 責務

### テスト設計

- テストケースを仕様から網羅的に洗い出す
- 境界値分析・同値分割を適用する
- テストの独立性と再現性を保証する

### 単体テスト

- Vitest で関数・サービス単位のテストを書く
- モック・スタブの使い分けを適切に行う
- カバレッジ目標を意識する

### 統合テスト

- API エンドポイントの E2E テストを実装する
- テストデータのセットアップ・クリーンアップを管理する
- MSW を活用した外部 API のモックを構築する

## 出力フォーマット

```text
## テスト計画

- 対象: テスト対象のモジュール
- テスト種別: 単体/統合
- テストケース一覧:
  - 正常系: ケース名 → 期待結果
  - 異常系: ケース名 → 期待結果
  - 境界値: ケース名 → 期待結果
- モック対象: 外部依存の一覧
```text
