# IaC 設計（Infrastructure as Code）

## 概要

インフラストラクチャをコードで管理し、再現性と一貫性を保証するスキル。

## 責務

### テンプレート設計

- AWS CDK/CloudFormation でインフラを定義する
- スタック分割の粒度を適切に設計する
- パラメータ化と環境分離を実現する

### 状態管理

- インフラ状態の一貫性を保つ
- ドリフト検知と修正を行う
- 安全なロールバック手順を用意する

### ベストプラクティス

- タグ付け戦略を統一する
- シークレット管理を AWS Secrets Manager/Parameter Store で行う
- リソースの命名規則を統一する

## 出力フォーマット

```text
## インフラ構成提案

- リソース: AWS サービス名
- 構成: スタック構成図
- パラメータ: 環境変数・設定値
- 依存関係: リソース間の依存
- コスト見積: 月額概算
```text
