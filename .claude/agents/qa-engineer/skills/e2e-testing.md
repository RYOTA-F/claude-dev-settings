# E2E テスト（End-to-End Testing）

## 概要

Playwright を用いた E2E テストの設計・実装を行うスキル。

## 責務

### テストシナリオ設計

- クリティカルなユーザーフローを特定する
- テストの実行順序と依存関係を管理する
- 並列実行可能な設計にする

### Playwright 実装

- Page Object Model でテストコードを構造化する
- ロケータ戦略を標準化する
- リトライとタイムアウトの設定を最適化する

### 安定性確保

- Flaky テストの検知と修正を行う
- テスト環境の安定性を維持する
- ビジュアルリグレッションテストを導入する

## 出力フォーマット

```text
## E2E テスト設計

- 対象フロー: ユーザーフロー名
- ステップ:
  1. ページ遷移 → アサーション
  2. 操作 → アサーション
- テストデータ: 必要なデータ
- 環境要件: テスト実行環境
- 実行時間: 目標時間
```text
