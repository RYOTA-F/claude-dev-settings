# プロトタイプ仕様（Prototype Spec）

## 概要

画面仕様書の作成、インタラクション定義、状態遷移設計を行い、実装前の設計品質を高めるスキル。

## 責務

### 画面仕様書作成

- 画面ごとにレイアウト・コンポーネント配置を定義する
- 表示するデータと動的要素を明記する
- レスポンシブ対応のブレークポイント別レイアウトを設計する
- エッジケース（データなし、長文、エラー）の表示を定義する

### インタラクション定義

- ユーザーアクション（クリック、ホバー、スワイプ等）ごとの振る舞いを定義する
- アニメーション・トランジションの種類と時間を指定する
- フィードバック（ローディング、成功、エラー）の表示方法を定義する

### 状態遷移設計

- 画面の状態（初期、ローディング、データあり、データなし、エラー）を列挙する
- 状態間の遷移条件を定義する
- 各状態での UI の見え方を明記する
