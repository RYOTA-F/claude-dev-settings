# セキュリティ監査（Security Audit）

## 概要

プロダクトの脆弱性を検出し、セキュリティリスクを低減するための対策を立案するスキル。

## 責務

### 脆弱性スキャン

- OWASP Top 10 に基づくチェックを実施する:
  - インジェクション（SQL、XSS、コマンド）
  - 認証・認可の不備
  - 機密データの露出
  - セキュリティの設定ミス
  - 既知の脆弱性を持つコンポーネント
- 依存ライブラリの脆弱性を定期的にスキャンする

### セキュリティレビュー

- 新機能・変更のセキュリティ観点でのレビューを行う
- レビュー観点:
  - 入力バリデーション
  - 認証・認可の実装
  - 機密情報の取り扱い
  - CORS・CSP の設定
  - API レート制限

### 対策立案

- 検出された脆弱性に対する修正方針を提案する
- セキュリティパッチの適用優先度を決定する
- 予防的なセキュリティ対策を提案する

## 出力フォーマット

```
## セキュリティ監査レポート
### サマリー
- 重大（Critical）: X 件
- 高（High）: X 件
- 中（Medium）: X 件
- 低（Low）: X 件

### 脆弱性一覧
| ID | 種別 | 重大度 | 対象 | 修正方針 | ステータス |
|----|------|--------|------|---------|-----------|

### 推奨アクション
1. [即時対応] ...
2. [計画的対応] ...
```
