# アーキテクチャレビュー（Architecture Review）

## 概要

システム設計のレビューを行い、適切なパターン選定と依存関係の健全性を維持するスキル。

## 責務

### 設計レビュー

- 新機能・大規模変更の設計を事前にレビューする
- レビュー観点:
  - 責務の分離が適切か
  - 拡張性・変更容易性が確保されているか
  - 既存アーキテクチャとの整合性があるか
  - テスタビリティが考慮されているか

### パターン選定

- 課題に対して適切なアーキテクチャパターンを提案する
- パターン選定の根拠をトレードオフとともに示す
- オーバーエンジニアリングを防ぎ、適切な抽象度を保つ

### 依存関係分析

- モジュール間の依存関係を可視化する
- 循環依存を検知し、解消方法を提案する
- 外部ライブラリの依存リスク（メンテナンス状況、ライセンス）を評価する
