# 技術負債トラッカー（Tech Debt Tracker）

## 概要

技術負債を可視化し、優先度を付けて計画的に返済するスキル。

## 責務

### 負債の可視化

- コードベースの技術負債を定期的にスキャンする
- 検出対象:
  - TODO / FIXME / HACK コメント
  - 非推奨 API の使用
  - テストカバレッジの低い重要モジュール
  - 複雑度の高い関数・クラス
  - 古いバージョンの依存ライブラリ

### 優先度付け

- 以下の基準でスコアリングする:
  - **影響度**: 放置した場合の悪化速度
  - **頻度**: 該当コードに触れる頻度
  - **修正コスト**: 修正に必要な工数
  - **リスク**: セキュリティ・安定性への影響
- スコアに基づいて返済の優先順位を決定する

### 返済計画

- スプリントごとに負債返済枠を確保する（全体の 15〜20%）
- 大規模な返済は段階的に実施する計画を立てる
- 返済の進捗と効果を追跡する

## 出力フォーマット

```
## 技術負債レポート
### サマリー
- 総負債項目: XX 件
- 高優先度: XX 件
- 前回比: +X / -X 件

### 負債一覧（高優先度）
| ID | 内容 | 影響度 | 修正コスト | スコア | ステータス |
|----|------|--------|----------|--------|-----------|

### 今期返済計画
1. [負債 ID]: 対応方針 → 担当 → 期限
```
