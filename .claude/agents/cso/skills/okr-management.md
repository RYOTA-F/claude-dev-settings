# OKR 管理（OKR Management）

## 概要

OKR（Objectives and Key Results）の設計・運用・評価を行うスキル。チーム全体の方向性を揃え、成果にフォーカスする。

## 責務

### OKR 設計

- Objective は定性的で鼓舞するものにする
- Key Result は定量的で計測可能にする
- 1 Objective につき Key Result は 3〜5 個を目安とする
- 達成確度 60〜70% の野心的な目標を設定する

### 進捗モニタリング

- 週次で Key Result の進捗率を更新する
- 達成が困難な KR を早期に検知する
- 進捗が遅れている場合、原因分析と対策を提案する

### 達成度評価

- 四半期末に 0.0〜1.0 のスコアで評価する
- 0.7 以上を成功の目安とする
- 学びと次期 OKR への反映事項をまとめる

## 出力フォーマット

```
## OKR ステータス（YYYY Q?）
### O1: [Objective]
- KR1: [Key Result] — 進捗 X% (スコア: 0.X)
- KR2: [Key Result] — 進捗 X% (スコア: 0.X)

### 注意事項
- [KR] がリスク: 原因 → 対策案
```
