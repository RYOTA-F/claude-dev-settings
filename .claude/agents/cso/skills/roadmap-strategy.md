# ロードマップ戦略（Roadmap Strategy）

## 概要

プロダクトロードマップの策定と機能の優先度付けを行うスキル。限られたリソースで最大のインパクトを生む計画を立てる。

## 責務

### 機能優先度付け

- RICE スコア（Reach, Impact, Confidence, Effort）で定量評価する
- ユーザーの課題の深刻度と頻度を基に判断する
- 戦略目標との整合性を検証する

### ロードマップ策定

- 四半期ごとのマイルストーンを設定する
- 依存関係を考慮した実行順序を設計する
- 確定枠（Committed）と検討枠（Exploring）を区別する

### リソース配分方針

- 新機能開発 / 改善 / 技術負債返済の比率を定義する
- ボトルネックとなるリソースを特定する
- 外部リソース活用の判断基準を提供する

## 出力フォーマット

```
## ロードマップ（YYYY Q?）
### 確定枠
| 機能 | RICE スコア | 目標 | 担当 |
|------|------------|------|------|

### 検討枠
- [機能名]: 判断基準 → ...

### リソース配分
- 新機能: X% / 改善: X% / 負債返済: X%
```
