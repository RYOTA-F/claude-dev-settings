# リスクレーダー（Risk Radar）

## 概要

プロジェクトのリスクを継続的に検知・評価し、障害発生時のトリアージを行うスキル。

## 責務

### リスク検知

以下の観点でリスクを継続的にスキャンする:

#### 技術リスク

- 依存パッケージの脆弱性（`npm audit` / `yarn audit`）
- 非推奨 API の使用
- ライセンス互換性の問題
- パフォーマンス劣化の兆候

#### 運用リスク

- 単一障害点（SPOF）の存在
- テストカバレッジの低い重要パス
- ドキュメントと実装の乖離
- 長期間メンテされていないコード

### リスク評価

検知したリスクを以下のマトリクスで分類する:

| | 影響: 低 | 影響: 高 |
| --- | --- | --- |
| **発生確率: 高** | 中（計画的に対処） | 緊急（即時対処） |
| **発生確率: 低** | 低（監視継続） | 中（対策を準備） |

### インシデント対応

障害発生時のフローを定義する:

1. **検知** — エラーログ・アラート・ユーザー報告から事象を確認
2. **トリアージ** — 影響範囲と緊急度を判定
3. **対処** — 応急処置（ロールバック・機能無効化等）を実行
4. **原因分析** — 根本原因を特定
5. **再発防止** — テスト追加・監視強化等の対策を実施

### リスクレポート

```text
## リスクサマリー

- 緊急: X 件
- 中: X 件
- 低: X 件

## 新規検知

- [リスク内容]（分類: 緊急 / 中 / 低）
  - 影響: ...
  - 推奨対応: ...
```text
