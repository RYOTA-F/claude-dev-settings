# プロセス改善（Process Kaizen）

## 概要

開発プロセスを継続的に改善するスキル。振り返りから学びを抽出し、ワークフローの最適化と自動化を推進する。

## 責務

### 振り返り分析

スプリント・リリース・インシデントの後に振り返りを実施する:

#### 分析フレームワーク
- **Keep** — うまくいったこと。継続する
- **Problem** — 問題があったこと。原因を深掘る
- **Try** — 次に試すこと。具体的なアクションに落とす

#### 深掘りの手法
- **5 Why** — 表層の問題から根本原因まで掘り下げる
- **タイムライン分析** — 何がいつ起きたかを時系列で整理する

### ワークフロー最適化

以下の観点で改善ポイントを特定する:

- **待ち時間の削減** — レビュー待ち、承認待ち、ビルド待ちを短縮する
- **手戻りの削減** — 手戻りが発生しやすい工程を特定し、上流で防ぐ
- **並列化** — 直列になっている独立した作業を並列実行に変更する
- **標準化** — 属人的な作業を手順化・テンプレート化する

### 自動化提案

繰り返し発生する手作業を自動化の候補として提案する:

#### 自動化の判断基準
- 頻度: 週 2 回以上実行されるか
- 定型度: 手順が毎回ほぼ同じか
- リスク: 手作業によるミスが発生しうるか

#### 自動化の優先度
1. CI/CD パイプラインの改善
2. テスト自動化の拡充
3. コード生成・スキャフォールディング
4. レポート・通知の自動化

### 改善レポート

```
## 改善サマリー

### 実施済み
- [改善内容]（効果: ...）

### 提案中
- [改善内容]
  - 期待効果: ...
  - 実装コスト: S / M / L
  - 優先度: 高 / 中 / 低
```
