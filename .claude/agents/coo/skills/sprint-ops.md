# スプリント運用（Sprint Ops）

## 概要

スプリントの計画・実行・監視を担うスキル。タスクの粒度を適切に保ち、進捗のボトルネックを早期に検知する。

## 責務

### タスク分解

- ユーザーストーリーを実装可能な粒度のタスクに分解する
- 各タスクに以下を明確にする:
  - 完了条件（Done の定義）
  - 依存関係（ブロッカーの有無）
  - 推定規模（S / M / L）
- 1 タスク = 1 PR を原則とする

### 進捗トラッキング

- アクティブなタスク一覧を定期的に確認する
- 以下の状態を監視:
  - 長時間 running のまま動きがないタスク
  - waiting_confirmation が放置されているタスク
  - 依存関係でブロックされているタスク
- 停滞を検知したら原因を特定し、対処を提案する

### ボトルネック検知

- 同一リソース（ファイル・API・DB テーブル）への競合を検知する
- タスク間の暗黙の依存関係を洗い出す
- 並列実行可能なタスクを特定し、スループットを最大化する

## 出力フォーマット

### 日次サマリー

```
## スプリント状況
- 完了: X / Y タスク
- 進行中: X タスク
- ブロック中: X タスク（原因: ...）

## 注意事項
- [タスク名] が N 時間停滞中 → 提案: ...
```
