# リリースパイロット（Release Pilot）

## 概要

リリースの計画・実行・監視を担うスキル。安全なデプロイと迅速なロールバックを保証する。

## 責務

### リリース計画

- リリース対象の変更一覧を作成する
- 破壊的変更の有無を確認する
- DB マイグレーションの有無と影響範囲を確認する
- リリースノートを作成する

### デプロイ手順

#### プリフライトチェック
1. 全テストが通過していること
2. DB マイグレーションがレビュー済みであること
3. 環境変数の追加・変更がドキュメント化されていること
4. 依存パッケージの脆弱性チェックが完了していること

#### デプロイフロー
1. staging 環境でのスモークテスト
2. カナリアリリース（対象を限定した本番反映）
3. メトリクス監視（エラー率・レイテンシ・リソース使用率）
4. 全面展開

### ロールバック判断

以下のいずれかに該当する場合、ロールバックを提案する:

- エラー率がデプロイ前の 2 倍を超えた
- P95 レイテンシがデプロイ前の 1.5 倍を超えた
- クリティカルな機能が動作していない
- DB マイグレーションに不整合が発生した

### ロールバック手順

1. トラフィックを旧バージョンに切り替え
2. 影響範囲を特定・記録
3. 根本原因の調査を開始
4. ステークホルダーへの報告
