# 品質ゲート（Quality Gate）

## 概要

コード品質の基準を定義し、マージ前のチェックを体系化するスキル。レビューの属人化を防ぎ、一貫した品質水準を維持する。

## 責務

### コードレビュー基準

レビュー時に以下の観点でチェックする:

#### 必須（ブロッカー）

- 型安全性: `any` の使用がないか
- セキュリティ: インジェクション・XSS・認証漏れがないか
- テスト: 新規ロジックにテストが追加されているか
- エラーハンドリング: 例外が適切に処理されているか

#### 推奨（警告）

- パフォーマンス: N+1 クエリ、不要な再レンダリングがないか
- 命名: 意図が伝わる命名になっているか
- 複雑度: 関数が長すぎないか（目安: 50 行以内）
- DRY: 明らかな重複がないか

#### 情報（参考）

- コードスタイルの一貫性
- コメントの過不足

### マージ判定

以下がすべて満たされた場合にマージ可とする:

1. 必須チェック項目にブロッカーがない
2. CI が全て通過している
3. テストカバレッジが既存水準を下回っていない
4. 推奨項目の警告が対応済み、または意図的なスキップとして記録されている

### 品質メトリクス

定期的に以下を計測・報告する:

- テストカバレッジの推移
- lint エラー数の推移
- PR あたりの指摘数の推移
- マージまでのリードタイム
