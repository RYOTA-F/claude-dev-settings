# テスト規約

## 基本方針

- テストフレームワークには Vitest を使用する
- テストファイルは対象ファイルと同階層に `*.test.ts` で配置する
- カバレッジ目標: ビジネスロジック 80% 以上

## テスト構造

- `describe` でテスト対象の関数・クラス名を記述する
- `it` でテストケースを日本語で記述する（例: `it("空文字列の場合はエラーを返す")`）
- AAA パターン（Arrange, Act, Assert）に従う

## テストの種類

### ユニットテスト
- 外部依存はモックする
- 1テストケース1アサーションを基本とする
- エッジケースと境界値を網羅する

### 統合テスト
- API エンドポイントのテストにはスーパーテストを使用する
- テスト用データベースを使用する（モックではなく実DB）
- テストごとにデータをクリーンアップする

### E2E テスト
- Playwright を使用する
- クリティカルパスのみをカバーする

## モック

- モックは必要最小限にとどめる
- 外部 API 呼び出しは MSW（Mock Service Worker）でモックする
- 日時のモックには `vi.useFakeTimers()` を使用する

## テストデータ

- ファクトリパターンを使用してテストデータを生成する
- ハードコードされたマジックナンバーを避ける
